Build with KAI.

Everything you need to integrate, control, and extend the KAI Memory Core and Voice Interface.

1. Python Backend API

KAI's backend runs on Python, allowing you to intercept voice intents and trigger custom scripts. Use the kai.api module to register custom commands.

import kai.api

# Register a custom voice command
def handle_custom_command(intent, entities):
    print(f"Received intent: {intent}")
    return "Command executed successfully"

kai.api.register_command(
    intent="change_background",
    callback=handle_custom_command
)

2. Memory Core Schema

The Memory Core stores data locally in a secure JSON format. You can read and manipulate this schema to sync memories across instances (coming in v2).

// ~/.kai/memory_core.json
{
  "identity": {
    "name": "User",
    "location": "Local"
  },
  "preferences": {
    "theme": "dark",
    "wake_word": "hey kai"
  },
  "recalls": [
    { "id": "r_001", "text": "Likes Python over JS" }
  ]
}

3. Electron UI Hooks

Customize the 3D avatar interface by hooking into KAI's Electron IPC events. Listen for state changes and update your frontend accordingly.

// renderer.js
window.kai.ipc.on('core:status', (event, data) => {
  if (data.status === 'listening') {
    triggerAvatarAnimation('listen');
  }
});
